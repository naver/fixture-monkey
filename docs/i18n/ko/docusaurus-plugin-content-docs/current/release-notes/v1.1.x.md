# v1.1.x

## v1.1.17

#### Features
- `JavaNodeTreeAdapterPlugin` 추가 — JvmNodeTree를 사용하는 대체 객체 생성 엔진
- `KotlinNodeTreeAdapterPlugin` 추가 — Node Tree Adapter의 Kotlin 지원
- `AdapterTracer` 추가 — 픽스처 생성 중 값 해석 과정을 디버깅하는 내장 트레이싱
- `object-farm-api` 모듈 추가 — JvmNodeTree, PathExpression, 타입 시스템 인프라

#### Node Tree Adapter 플러그인

제로 오버헤드와 내장 디버깅 기능을 갖춘 대체 객체 생성 경로를 제공하는 새 플러그인입니다.
**v1.2.0부터 Node Tree Adapter가 기본 객체 생성 엔진이 됩니다.**

**Java:**
```java
FixtureMonkey fixtureMonkey = FixtureMonkey.builder()
    .plugin(new JavaNodeTreeAdapterPlugin())
    .build();
```

**Kotlin:**
```kotlin
val fixtureMonkey = FixtureMonkey.builder()
    .plugin(KotlinPlugin())
    .plugin(KotlinNodeTreeAdapterPlugin())
    .build()
```

**벤치마크 결과** (JMH, fork=1, warmup=3, iterations=10, iteration당 200개 OrderSheet 객체 생성):

| 시나리오 | Adapter 미사용 | Adapter 사용 | 오버헤드 |
|---------|--------------|-------------|---------|
| 순수 sample | 125.2 ms | 121.4 ms | -3.1% |
| set | 121.7 ms | 120.7 ms | -0.8% |
| size + setNull | 122.6 ms | 123.4 ms | +0.6% |
| 중첩 set | 124.1 ms | 124.8 ms | +0.5% |
| thenApply | 333.3 ms | 301.1 ms | **-9.7%** |

모든 시나리오에서 오버헤드가 거의 없습니다. `thenApply`는 adapter 사용 시 일관되게 ~10% 빠릅니다.

**AdapterTracer를 사용한 디버깅:**
```java
FixtureMonkey fixtureMonkey = FixtureMonkey.builder()
    .plugin(new JavaNodeTreeAdapterPlugin()
        .tracer(AdapterTracer.console()))
    .build();
```

자세한 내용은 [Node Tree Adapter 플러그인 문서](/docs/plugins/node-tree-adapter-plugin/features)를 참조하세요.
