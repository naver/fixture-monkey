#### Node Tree Adapter Plugin

A new plugin that provides an alternative object generation path with zero overhead and built-in debugging capabilities.
**Starting from v1.2.0, the Node Tree Adapter will become the default object generation engine.**

**Java:**
```java
FixtureMonkey fixtureMonkey = FixtureMonkey.builder()
    .plugin(new JavaNodeTreeAdapterPlugin())
    .build();
```

**Kotlin:**
```kotlin
val fixtureMonkey = FixtureMonkey.builder()
    .plugin(KotlinPlugin())
    .plugin(KotlinNodeTreeAdapterPlugin())
    .build()
```

**Benchmark Results** (JMH, fork=1, warmup=3, iterations=10, 200 OrderSheet objects per iteration):

| Scenario | Without Adapter | With Adapter | Overhead |
|----------|----------------|-------------|----------|
| Pure sample | 125.2 ms | 121.4 ms | -3.1% |
| set | 121.7 ms | 120.7 ms | -0.8% |
| size + setNull | 122.6 ms | 123.4 ms | +0.6% |
| Nested set | 124.1 ms | 124.8 ms | +0.5% |
| thenApply | 333.3 ms | 301.1 ms | **-9.7%** |

All scenarios show near-zero overhead. `thenApply` is consistently ~10% faster with the adapter.

**Debugging with AdapterTracer:**
```java
FixtureMonkey fixtureMonkey = FixtureMonkey.builder()
    .plugin(new JavaNodeTreeAdapterPlugin()
        .tracer(AdapterTracer.console()))
    .build();
```

See [Node Tree Adapter Plugin documentation](/docs/plugins/node-tree-adapter-plugin/features) for details.
